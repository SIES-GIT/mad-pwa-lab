# Experiment 10: Analyze Web Application using Google Lighthouse

## Aim
To analyze and evaluate the performance, accessibility, best practices, and SEO of a web application using Google Lighthouse.

---

## Theory

Google Lighthouse is an open-source, automated tool developed by Google to analyze and improve the quality of web pages.

It audits a website across multiple dimensions such as:
- Performance
- Accessibility
- Best Practices
- SEO

It generates a detailed report with scores and suggestions for improvement.

---

## Tools Required

- Google Chrome Browser
- Internet Connection

---

## Steps to Perform Experiment

### Step 1: Open Web Application

Open the deployed website in Chrome browser:

https://www.canva.com/create-new
0r (any website link )

---

### Step 2: Open Developer Tools

1. Right click on the page
2. Click on "Inspect"
3. Go to the "Lighthouse" tab

---

### Step 3: Configure Lighthouse

Select the following categories:
- Performance
- Accessibility
- Best Practices
- SEO

---

### Step 4: Generate Report

1. Click on "Analyze page load"
2. Wait for analysis to complete

---

### Step 5: View Report

Lighthouse generates a report showing scores and suggestions for improvement.

---

## Output

The Lighthouse report displays:

- Performance score
- Accessibility score
- Best Practices score
- SEO score

Each score is given on a scale of 0 to 100:
- 90–100: Good
- 50–89: Needs improvement
- 0–49: Poor

The tool also provides suggestions to improve the web application.

---

## Conclusion

The experiment was successfully performed using Google Lighthouse, providing insights into performance, accessibility, best practices, and SEO, along with suggestions for improvement.
